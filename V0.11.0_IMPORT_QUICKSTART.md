# v0.11.0 AI Knowledge Import Quickstart

## 1. Create the support platform first

Open **Admin → AI Knowledge Import**. In **Support platform profiles**, create one profile for each app or website.

Choose the support mode that matches that platform:

| Mode | Use it when | Ticket buttons |
| --- | --- | --- |
| `No ticket system` | Members contact support directly and the app has no ticket workflow. | Hidden automatically |
| `Tickets` | Members should create support tickets. | Allowed |
| `Hybrid` | Members can use tickets or a normal support route. | Allowed |

Use a simple stable key, for example `bdg-india` or `bdg-vip`. Never use `all` because it is reserved.

## 2. Prepare the Excel workbook

Upload an `.xlsx` workbook. Required columns are:

| Column | Purpose |
| --- | --- |
| `Question` | The customer question or topic. |
| `How to reply/Answer` | The approved answer used to create the knowledge draft. |

Optional columns include `Photo/Picture`, `Corresponding Ticket`, `Positive examples`, `Negative examples`, `AI instruction`, `Locale`, and `Platform`.

`Photo/Picture` is a review note. Add the actual image manually in **AI Prompt & Image → Visual knowledge editor** after the draft is created. `Corresponding Ticket` is also a review label; it never creates a ticket automatically.

## 3. Import safely

1. Select the target platform and upload the workbook.
2. Review the created batch. Fix any rows marked with an error.
3. Click **Create AI Content drafts**.
4. Open each new draft in **AI Prompt & Image**.
5. Add or refine instructions, positive and negative examples, visual knowledge, and only the approved buttons.
6. Set **Knowledge approval** to `Approved` and **Status** to `Published`.

Only approved and published content can be considered by AI. Importing a workbook never changes live answers by itself.

## 4. Configure buttons

In **Admin → Buttons configuration**, set the platform scope and capability for every button:

- `General`: ordinary app or web links.
- `Support`: normal support links.
- `Ticket`: shown only when the selected platform mode is `Tickets` or `Hybrid`.

The AI may only show buttons you attached to the approved knowledge item. It cannot invent ticket links.

## 5. Use the right platform in public links

Append a platform key to the public URL when a page belongs to a specific app:

```text
https://bdg-chat-pages.pages.dev/?platform=bdg-india
https://bdg-guide-pages.pages.dev/?platform=bdg-india
```

Without the parameter, the `default` platform is used.
