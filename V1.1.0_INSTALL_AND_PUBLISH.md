# v1.1.0 install and publish

## One simple install step

1. Extract the v1.1.0 release zip.
2. Double-click `INSTALL-V1.1.0-TENANT-DATA-ISOLATION.cmd`.
3. Wait for the green `INSTALL COMPLETE` message, then press a key.

The installer only copies the release files into this exact repository:

`C:\Users\LENOVO\Documents\cloud-projects\BDG_CS_ASSISTANT`

It does **not** use PowerShell, npm, Cloudflare, Render, GitHub, or any
Myanmar-2D repository. It stops safely if this exact directory is not the
`deathnotelaymm-maker/BDG_CS_ASSISTANT` Git repository or if it has uncommitted
work.

## Publish with GitHub Desktop

1. Open GitHub Desktop and select **BDG_CS_ASSISTANT**.
2. Review the listed changes.
3. Enter the summary: `v1.1.0 Tenant Data Isolation & Platform-Scoped Admin`.
4. Click **Commit to main**, then **Push origin**.
5. Wait for Render to show the matching v1.1.0 release. Your existing GitHub
   workflow publishes the Admin, Chat, and Guide Pages apps.

## Validate after the automated deployment

- Open a generated platform Admin URL.
- Confirm the header shows `Platform: <name>`.
- Create a test FAQ or guide only inside that platform.
- Open a different platform URL and confirm that test item is not visible.
- Confirm `/chat` and `/guide` use the generated platform URL, not the legacy
  default pages.

## Important

Do not create custom domains in this workflow. Generated paths work on your
existing Pages domains now; custom domains can be mapped later in Cloudflare.
